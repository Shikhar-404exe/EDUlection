# DemocracyOS Lite — Core Definition

## 🎯 Goal

Build a lightweight Civic Intelligence System that helps users:

* Understand election processes
* Explore timelines
* Simulate outcomes
* Detect misinformation
* Interact via roles

---

## 🧠 Core Modules

1. Chat Assistant (Hinglish)
2. Timeline Viewer
3. Scenario Simulator (with Chart.js)
4. Misinformation Checker
5. Role Selector (Voter / Officer)

---

## 🌍 Hinglish Mode

* Default language: Hinglish (Hindi + English mix)
* Tone: conversational and simple
* Example:
  "Voting process simple hai — pehle registration hota hai, phir aap vote cast karte ho"

---

## 🚫 STRICT CONSTRAINTS

### ❌ DO NOT USE:

* Firebase
* MongoDB
* Any database
* Authentication systems
* Real election APIs
* Government APIs
* Kubernetes
* Microservices
* Complex routing
* Redux or heavy state libraries
* Animation libraries

---

## ✅ ALLOWED APPROACH

* Static frontend (HTML + Tailwind CDN)
* Vanilla JavaScript
* Chart.js (CDN)
* LLM for reasoning (added later)

---

## 🧠 GOLDEN RULES

* Keep everything minimal
* Prefer prompts over code
* Use mock data instead of real APIs
* Focus on working prototype, not production system
